Códigos síncrono e assíncrono

Em JavaScript, os termos "síncrono" e "assíncrono" descrevem como o código é executado. A principal diferença entre eles reside na forma como o fluxo de execução é gerenciado.


Código síncrono

No código síncrono, as tarefas são executadas sequencialmente, uma após a outra. Cada instrução deve ser concluída antes que a próxima possa começar. Imagine uma fila de pessoas em um caixa de supermercado: cada pessoa só é atendida depois que a anterior termina.


Código assíncrono

No código assíncrono, as tarefas podem ser executadas de forma independente, sem bloquear o fluxo principal de execução. Imagine um restaurante: enquanto um cliente espera seu pedido, outros clientes podem ser atendidos.
